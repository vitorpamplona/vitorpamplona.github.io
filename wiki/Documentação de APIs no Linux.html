Documentação de APIs no Linux

Já tentaram programar off-line? É, eu sei. Hoje em dia todo mundo tem internet, para que pensar em off-line? Alguns tem até wireless para levar o notebook onde quiser. Mas esse não é o meu caso. Levo o notebook para alguns lugares remotos que, mesmo se lá houvesse internet, seria de baixa qualidade. Como, por exemplo, o sofá do meu AP: D.

Estou programando off-line em várias linguagens. Algumas eu ainda não domino e, portanto, preciso acessar a documentação da API frequentemente. Como vivo mudando de IDE, a documentação presente nelas nem sempre é a mesma, muito menos formatos, atalhos e etc. Quando estou on-line, basta digitar o nome da classe ou método e o google resolve o meu problema, porém, quando estou off-line, o bixo pega.

Ontem resolvi baixar vários pacotes de documentação (* - doc) do Ubuntu para tentar resolver o meu problema. Apesar de instalar com sucesso, cada pacote tem sua própria estrutura de documentação, nem o caminho para o " index.html " é parecido. Alguns você instala e se pergunta: " Tah, e agora, onde foi parar essa documentação? ". Pensei em instalar o Google Desktop para pesquisar nos arquivos de documentação com mais qualidade, mas isso é como matar uma vaca com um F16.

Depois de uma vasculhada na internet, descobri o DevHelp, um aplicativo cujo objetivo é reunir documentações, facilitando a leitura e as buscas. Justamente o que eu precisava, um aplicativo externo, simples, fácil e bem leve. Mas obviamente ele não resolveu o meu problema, pois ele só lê documentações em um formato, e poucas bibliotecas usam este formato. Python e GTK são dois exemplos que usam, mas ruby, java e c + + não usam, infelizmente.

Por isso, aproveitei o Ubuntu Brainstorm e mandei lá uma idéia para eles: Aumentar o suporte para o DevHelp. Se você tem esse mesmo problema, vote lá! Mesmo assim, estou pensando em mandar alguns patches para incluir o arquivo do DevHelp na documentação de alguns projetos, se aceitarem, já fico feliz.
